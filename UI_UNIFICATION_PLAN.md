# UI 统一执行计划

更新时间：2026-06-30

本文档用于严格执行项目 UI 统一工作。设计标准以 `PROJECT_UI_AND_WORKFLOW_STANDARD.md` 为准，本计划只规定执行顺序、验收方式和当前进度。

## 执行原则

- 先统一基础变量，再统一组件，再统一页面。
- 每一轮只做低耦合、可验证的改动。
- 不借 UI 统一改业务功能。
- 不一次性重写大面积页面结构。
- 不覆盖与当前任务无关的已有改动。
- 每一轮完成后必须记录完成项、检查结果和下一轮任务。

## 当前对话升级流程

以后在当前项目中继续 UI 规范化时，默认按以下流程执行：

1. 读取 `PROJECT_UI_AND_WORKFLOW_STANDARD.md`，确认设计标准。
2. 读取本文档，确认当前 UI 规范化轮次。
3. 先判断本次任务属于第几层。
4. 只执行当前层级中最小可验证的一批改动。
5. 不改业务功能，不改变数据逻辑。
6. 完成后更新本文档的进度。
7. 运行项目检查。
8. 在最终回复中说明改动范围、检查结果和下一步。

如果用户只说“继续下一步”“继续 UI 规范化”“按规范执行”，默认从本文档的“下一轮”继续推进。

## 分层路线

### 第 1 层：设计变量统一

目标：把颜色、圆角、间距、控件高度、聚焦环、阴影等基础值沉淀为变量。

执行项：

- [x] 确认设计规范文档。
- [x] 增加基础 UI token：圆角、间距、控件高度、聚焦环、动效。
- [ ] 逐步减少散落的硬编码颜色。
- [ ] 逐步减少散落的硬编码圆角和高度。

验收标准：

- 新增样式优先使用变量。
- 主按钮、次按钮、卡片、输入框可从统一变量读取基础尺寸。
- 不影响现有页面功能。

### 第 2 层：基础组件统一

目标：统一最常用的 UI 控件。

执行项：

- [ ] 按钮：主按钮、次按钮、危险按钮、图标按钮。
- [ ] 输入控件：输入框、搜索框、下拉框、日期输入。
- [ ] 容器：卡片、面板、弹窗、表格容器。
- [ ] 状态：hover、active、focus、disabled、loading。
- [ ] 标签：状态标签、分类标签、结果数量标签。

验收标准：

- 同类按钮高度一致。
- 同类输入框高度一致。
- focus 状态清晰统一。
- 禁用状态明显不可点击。
- 移动端点击区域不小于基础规范。

### 第 3 层：全局布局统一

目标：统一导航、顶部区、页面标题区、工具栏、内容容器。

执行项：

- [ ] 统一页面标题区间距。
- [ ] 统一工具栏布局。
- [ ] 统一内容区最大宽度和栅格。
- [ ] 统一卡片列表间距。
- [ ] 统一表格和列表容器。

验收标准：

- 页面结构视觉一致。
- 主操作位置稳定。
- 搜索、筛选、排序在统一工具区。
- 宽屏不拉散，窄屏不溢出。

### 第 4 层：移动端统一

目标：所有页面在常见移动尺寸下稳定可用。

执行项：

- [ ] 检查 320px、375px、390px、414px、768px。
- [ ] 消除页面级横向滚动。
- [ ] 表格在移动端改为卡片或局部滚动。
- [ ] 弹窗适配窄屏。
- [ ] 工具栏和筛选区移动端不占用过高首屏。

验收标准：

- 页面不出现横向滚动。
- 按钮和输入框不溢出。
- 长文本不撑破容器。
- 弹窗和菜单不会超出屏幕。

### 第 5 层：状态和边界统一

目标：补齐成熟项目需要的 UI 状态。

执行项：

- [ ] 空状态。
- [ ] 加载状态。
- [ ] 错误状态。
- [ ] 成功反馈。
- [ ] 危险操作确认。
- [ ] 长文本和异常数字。
- [ ] 权限不足和网络失败提示。

验收标准：

- 用户操作后有明确反馈。
- 错误信息说明原因和下一步。
- 危险操作不会误触。
- 空页面不是“坏掉”的视觉感受。
