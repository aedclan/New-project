## 一、升级目标

将当前 Personal Hub 从“功能逐步堆叠的个人管理网页”升级为“Opera One 风格的个人生活工作台”。

本次 UI 升级不追求表面模仿 Opera，而是参考它的核心设计语言：

- 左侧高效侧栏
- 模块化工作区
- 深色高级界面
- 清晰的信息层级
- 轻量动效
- 高密度但不拥挤的布局
- 快速切换、快速录入、快速查看

最终效果应该是：网站看起来更统一、更现代、更有设计感，同时不牺牲当前已有的账单、订阅、人情往来、事项、笔记、设置等功能。

## 二、设计定位

产品定位：

> 一个 Opera 风格的个人 / 家庭数据工作台，用来管理生活收支、订阅、人情往来、事项、笔记、收藏、账号同步和数据安全。

设计关键词：

- 深色
- 克制
- 高级
- 模块化
- 高效率
- 有秩序
- 信息密度合理
- 轻动效

避免方向：

- 不做电竞风。
- 不做大面积霓虹渐变。
- 不做满屏彩色卡片。
- 不做过度圆角和厚重阴影。
- 不做营销官网式大横幅。
- 不做只好看但难操作的界面。

## 三、视觉风格规范

### 1. 主色系统

```text
页面背景：#0f1014
侧栏背景：#15161c
主内容背景：#11131a
卡片背景：#1b1d25
二级卡片背景：#222530
悬停背景：#272a36
边框颜色：#2c2f3a
主文字：#f4f5f7
次文字：#9ca3af
弱文字：#6b7280
```

### 2. 强调色

```text
Opera 红：#ff1b2d
数据蓝：#5b8cff
成功绿：#30d158
警告橙：#ffb020
危险红：#ff4d5e
```

使用规则：

- Opera 红只用于当前选中、主要操作、重要提醒。
- 蓝色用于同步、图表、数据趋势。
- 橙色用于临期、待处理。
- 红色用于逾期、错误、危险操作。
- 不允许每个模块随意使用不同主色。

### 3. 圆角和边框

```text
小按钮：8px
输入框：10px
普通卡片：12px
弹窗：16px
侧栏按钮：10px
```

边框统一使用低对比细线：

```css
border: 1px solid var(--line);
```

### 4. 阴影

阴影只用于弹窗、悬浮面板、当前高亮卡片。

普通卡片不使用厚重阴影，主要通过背景层级和边框区分。

### 5. 字体层级

```text
页面标题：22px - 26px
模块标题：18px - 20px
卡片标题：15px - 17px
正文：14px
辅助信息：12px - 13px
数据数字：20px - 28px
```

原则：

- 标题不能过大。
- 数据数字可以突出。
- 表单文字要清晰。
- 按钮文字不能拥挤。

## 四、全局布局规划

### 1. 页面结构

升级为三层结构：

```text
左侧侧栏 + 顶部状态栏 + 主工作区
```

### 2. 左侧侧栏

侧栏参考 Opera 的垂直工具栏。

固定入口：

1. 总览
2. 生活收支
3. 订阅
4. 人情往来
5. 事项
6. 笔记
7. 收藏
8. 设置

侧栏要求：

- 默认窄宽度，图标为主。
- 鼠标悬停或展开时显示文字。
- 当前页面有红色竖线或红色背景点提示。
- 底部显示账号状态、同步状态、版本号。
- 移动端改为底部导航或抽屉导航。

### 3. 顶部状态栏

顶部状态栏展示：

- 当前模块名称
- 当前登录账号
- 同步状态
- 全局搜索
- 快捷新增按钮

要求：

- 高度控制在 56px - 64px。
- 不放太多文字。
- 账号名称要明显，但不能抢主内容。

### 4. 主工作区

每个功能模块都是独立工作区，不再让页面信息互相挤压。

每个工作区统一结构：

```text
页面标题区
核心指标区
主要内容区
辅助内容区
```

## 五、组件统一规范

### 1. 按钮

按钮分为四类：

```text
主按钮：红色，用于保存、新增、确认。
次按钮：深色描边，用于取消、筛选、普通操作。
图标按钮：用于编辑、删除、展开、关闭。
危险按钮：红色边框或红色文字，用于删除、清空。
```

按钮规则：

- 同一区域最多一个主按钮。
- 删除、清空必须有二次确认。
- 小操作优先使用图标按钮。
- 不再使用大小不一的按钮混排。

### 2. 输入框

输入框统一：

- 深色背景
- 低对比边框
- 聚焦时蓝色或红色细边
- 高度统一为 40px - 44px

搜索框要求：

- 圆角更明显。
- 左侧搜索图标。
- 输入时不跳转页面。
- 支持清空按钮。

### 3. 卡片

卡片分为：

```text
指标卡片
人物卡片
订阅卡片
账单记录卡片
笔记卡片
设置卡片
```

卡片规则：

- 统一背景、边框、圆角。
- 信息从上到下排列，不横向乱跳。
- 重要数字靠右或单独突出。
- 卡片之间间距统一。
- 卡片 hover 时轻微抬升或边框变亮。

### 4. 弹窗

所有弹窗统一：

```text
顶部：标题 + 关闭按钮
中部：内容区
底部：操作按钮
```

弹窗规则：

- 宽度按内容分级。
- 编辑表单不贴边。
- 多字段表单使用两列布局。
- 移动端自动单列。
- 点击遮罩关闭详情弹窗，编辑类弹窗需谨慎关闭。

### 5. 标签

标签用于：

- 关系类型
- 事件类型
- 状态
- 分类
- 筛选条件

标签要求：

- 统一高度。
- 统一圆角。
- 选中态明显。
- 不使用过多颜色。

## 七、动效规划

动效原则：

- 轻
- 快
- 不干扰操作
- 不影响性能

建议动效：

- 侧栏展开：160ms
- 卡片 hover：120ms
- 弹窗出现：160ms
- 页面切换：淡入 120ms
- 数据刷新：轻微 shimmer 或状态点变化

禁止动效：

- 大面积旋转
- 复杂粒子
- 长时间加载动画
- 每个卡片不同动画

## 八、响应式规划

### 桌面端

- 左侧固定侧栏。
- 顶部固定状态栏。
- 主内容区使用 12 栅格。
- 卡片根据内容自动 2 列、3 列或 4 列。

### 平板端

- 侧栏可收起。
- 主内容区 2 列为主。
- 表格转为紧凑列表。

### 手机端

- 侧栏改底部导航或抽屉。
- 主内容单列。
- 弹窗全屏或接近全屏。
- 表单单列。

## 九、执行优先级

优先级从高到低：

1. 全局框架
2. 组件统一
3. 首页总览
4. 人情往来
5. 生活收支
6. 订阅管理
7. 事项和笔记
8. 设置与部署

原因：

- 先统一框架，后续每个模块不会重复返工。
- 人情往来和生活收支是当前最核心的数据模块。
- 订阅、事项、笔记可以在统一组件后逐步增强。

网站达到以下状态，可以认为 Opera 风格 UI 升级完成：

- 整体视觉统一。
- 左侧导航清晰。
- 顶部状态明确。
- 每个模块像独立工作区。
- 信息密度高但不杂乱。
- 核心数据一眼可见。
- 表单和弹窗操作顺畅。
- 移动端可正常使用。
- 登录、同步、导入导出、备份、通知位置清晰。
- 用户长时间使用不会觉得压迫、混乱或疲劳。
