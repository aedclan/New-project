# 邮箱注册与账号安全升级路线

更新时间：2026-06-27

目标：用户注册不再使用普通账号名，统一改为邮箱注册、邮箱登录，并为后续邮箱验证、找回密码、多用户安全管理打基础。

## 总体路线

1. V1 邮箱注册与邮箱登录基础版
2. V2 邮箱验证版
3. V3 管理员邮箱账号管理版
4. V4 旧账号迁移收尾版

## V1 邮箱注册与邮箱登录基础版

### 目标

注册时只能使用邮箱，不能再使用普通账号名。登录界面以邮箱为主，账号显示也优先显示邮箱。

### 已升级内容

- 登录窗口字段从“邮箱 / 管理员账号”改为“邮箱”。
- 注册表单使用 `email` 字段。
- 前端注册时会检查是否填写邮箱。
- 后端注册接口只读取 `email`，不再把普通账号名当成注册入口。
- 新注册用户的 `username` 和 `email` 都写入邮箱，便于兼容旧数据结构。
- 登录接口优先按邮箱查找用户。
- 后端提示文案改为正常中文。
- 账号卡片优先显示邮箱。

### 当前兼容策略

- 旧管理员账号仍可通过 `username` 兼容登录。
- 如果管理员环境变量 `PERSONAL_HUB_ADMIN_USERNAME` 配置为邮箱，新管理员账号会自动带邮箱。
- 旧用户如果 `username` 本身是邮箱，系统会自动补入 `email` 字段。

### 验收标准

- 注册窗口只能输入邮箱格式。
- 新用户注册必须使用邮箱。
- 已注册邮箱不能重复注册。
- 使用邮箱 + 密码可以登录。
- 登录后左侧账号卡片显示邮箱。
- 普通账号名不能作为新用户注册入口。

## V2 邮箱验证版

### 目标

注册后必须验证邮箱，防止乱填邮箱和无效账号。

### 已升级内容

- 新增邮箱验证 token 表。
- 注册后发送验证邮件。
- 验证链接默认有效期 24 小时。
- 用户点击邮件链接后完成验证。
- 未验证邮箱禁止登录。
- 登录页支持重新发送验证邮件。
- 验证邮件使用 Resend，复用 `RESEND_API_KEY`。

### 后端表结构规划

```text
email_verification_tokens
- token_hash
- user_id
- expires_at
- created_at
- used_at
```

用户表增加：

```text
email_verified
email_verified_at
```

## V3 管理员邮箱账号管理版

### 目标

管理员可以围绕邮箱管理用户。

### 已升级内容

- 用户列表显示邮箱、角色、验证状态、禁用状态。
- 管理员可以重置密码。
- 管理员可以禁用 / 启用用户。
- 管理员可以重新发送验证邮件。
- 管理员可以手动标记邮箱已验证。
- 管理员可以给旧用户补邮箱或修改邮箱。
- 旧数据迁移时按邮箱识别用户。

## V4 旧账号迁移收尾版

### 目标

逐步废弃无邮箱旧账号，账号体系完全统一到邮箱。

### 已升级内容

- 无邮箱普通旧账号会被禁止登录，并提示联系管理员补充邮箱。
- 管理员可以给旧账号补邮箱。
- 管理员旧账号保留兼容登录，避免生产环境被锁死。
- 全站账号显示统一为邮箱或昵称。
- 文档和部署配置统一说明管理员账号建议使用邮箱。
- 增加忘记密码邮件重置。
- 增加登录页“忘记密码”入口。
- 增加设置页“修改我的密码”入口。
- 修改当前密码后会清除登录会话，要求重新登录。

## 环境变量建议

当前阶段：

```bash
PERSONAL_HUB_ADMIN_USERNAME=your-email@example.com
PERSONAL_HUB_ADMIN_PASSWORD=至少12位强密码
PERSONAL_HUB_REGISTRATION_ENABLED=false
PERSONAL_HUB_REGISTRATION_CODE=
```

后续邮箱验证阶段建议增加：

```bash
AUTH_EMAIL_FROM=Personal Hub <notice@example.com>
PUBLIC_SITE_URL=https://www.aedclan.com
AUTH_EMAIL_VERIFICATION_MAX_AGE=86400
AUTH_PASSWORD_RESET_MAX_AGE=3600
```

## 当前下一步

当前已推进到 V4 的第一轮可用形态。下一步建议：

1. VPS 配置 `RESEND_API_KEY`、`AUTH_EMAIL_FROM`、`PUBLIC_SITE_URL`。
2. VPS 配置管理员账号为邮箱。
3. 测试新邮箱注册、验证邮件、邮箱登录。
4. 管理员面板测试补邮箱、重发验证、标记已验证。
5. 测试忘记密码邮件重置。
6. 测试登录后修改我的密码。
